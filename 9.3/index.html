    Версия 9.3 (Декабрь 2021)

    В данной статье приведено подробное описание всех изменений, включенных в релиз ГИС ГеоАналитика версии 9.3, включающий в себя модернизацию Модуля веерных запросов для проработки услуг 941, 040 и 942, а также проверки возможности подключения по технологии FTTB.

    Модуль проработки веерных запросов

    Обновленный дизайн и оптимизированное расположение функциональных блоков

    В новой версии был значительно обновлен дизайн и повышено удобство использования основного функционала, оптимизирован путь для некоторых операций, в т.ч.:

    • Импорт вынесен в нижний блок, что позволяет одновременно смотреть результаты геокодирования и карту, а также выполнять навигацию к геокодированному адресу
    • Вкладки нижнего блока расположены горизонтально, что позволило более эффективно использовать горизонтальное место для таблиц
    • Тендеры сгруппированы, ряд информации объединен в одной ячейке также для более эффективного использования пространства. Закреплена первая колонка, что позволяет при перемотки вправо видеть глазами, к какому адресу относится результат
    • Исправлен ряд отступов для повышения читабельности текста в таблице
    • Проведен редизайн в соответствии с новым брендбуком

    Проверка услуг 941, 040, 942

    В новой версии модуля веерных запросов помимо 901 услуги, реализованной ранее в рамках версии 9.0, доступна проверка возможности подачи в точке услуг 941, 040 и 942. Соответствующие значения добавлены в выпадающий список формы импорта данных, а также могут быть добавлены в загружаемый Excel.

    Для 901 услуги, как и ранее, необходимо указать профиль, для услуг 941, 040 и 942 профиль не указывается, а опционально может быть задан ID вирутальной сети (в этом случае будет проверять только оборудование, поддерживающее режимы tagged и QinQ).

    Обратите внимание, что формат входного файла Excel был обновлен колонкой ID виртуальной сети. Новый шаблон может как и ранее быть скачан на первом шаге импорта.

    Проверка по FTTB

    Также в новой версии доступна возможность проверки возможности подключения услуг 901 и 941 по технологии FTTB.

    В рамках проверки возможности подключения по FTTB пространственная информация о ближайших точках коллективного доступа берется по данным соответствующего слоя из группы "Активное оборудование" (источник данных слоя - система Orange). Также c использованием сервисов ИНАК и FTTB проверяется отсутствие глобальной проблемы на адресе. В случае наличия ГП результатом проверки будет являться "Условная ТВ", требующая внимания проектировщика. Такой же результат устанавливается в случае технических проблем или расхождения данных при проверке отсутствия ГП.

    В результатах проверки ТВ (в т.ч. сохраненных) добавлена новая колонка "Средство подачи", которая указывает, относится ли найденное активное оборудование к подключениям OnNet посредством меди или же к подаче сервиса с использованием FTTB.

    Проверка ВИП-клиентов

    В новой версии доступна возможность проверки подачи сервиса для ВИП-клиентов. Для этого на вкладке "Активные запросы" необходимо перед проверкой ТВ установить флаг "ВИП-клиент", который активирует в рамках проверки дополнительные варианты подключения. По умолчанию, если галка не выбрана, то проверяется возможность подачи сервиса для бизнес-клиентов.

    Поиск нескольких ближайших точек

    В случае, если в искомом радиусе не было найдено подходящего оборудования, то в колонке "Ближайшая точка" в новой версии выводится информация о нескольких ближайших точках каждого типа.

    По умолчанию, как и ранее, радиус проверки для активного оборудования и ТКД составляет 200 метров, однако может быть изменен в рамках настройки системы.

    По умолчанию система выполняет поиск 3 ближайших точек по слоям IP-оборудования, РРЛ, МЕН и ТКД и выводит информацию по сайту для этого оборудования. Обращаем внимание, что несколько равноудаленных точек считаются за одну, т.е. поиск 3 ближайших точек может выдать больше 3 вариантов подключения, если расстояние до нескольких одинаково.

    Ручное заполнение результатов проектирования

    Для сохраненных ранее результатов проверки ТВ в статусе "Ручное проектирование" добавлена возможность ручного заполнения результатов проработки с помощью специальной формы.
